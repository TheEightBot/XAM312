Customizing the ListView in Xamarin.Forms - XAM312

ListView has a limited selection of built-in cell styles. These standard cells handle common cases and you should use them if they meet your needs. If not, you can create a custom cell that will uniquely represent your data and match the overall look-and-feel of your app. This course shows you how to define a row template, how to add headers/footers, and how to display grouped data. You will also survey several performance-tuning strategies including cell caching.

Upon completion you will be able to:

  • Create custom cell definitions
  • Add headers and footers to a ListView
  • Display grouped data in a ListView
  • Customize the cell view based on data
  • Optimize ListView Performance

Exercises

  1. Exercise 1: Provide a custom cell template for a ListView (XAM312)

    This exercise modifies an existing application that displays a list of animals. The app is using a ListView with an ImageCell to display the animals. You will replace the ImageCell with a custom ViewCell template to display additional information and provide additional formatting.

  2. Exercise 2: Add a header and footer to the ListView (XAM312)

    In this exercise, you'll add a header and footer definition to a ListView.

  3. Exercise 3: Add grouping support to a list (XAM312)

    In this exercise, you'll add grouping and sorting support to the ListView so that the animals are grouped by the first letter of their name.

  4. Exercise 4: Display different cells based on the bound-data (XAM312)

    In this exercise, you'll create and use a DataTemplateSelector to display different cells in a ListView based on the bound data. The application you'll use simulates a "smart home" dashboard that allows the user to see the status and interact with their smart home devices.
